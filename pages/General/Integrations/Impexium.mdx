## Features


**Single Sign On**: Slayte uses the Impexium accounts and records. No need to create new accounts.


**Embedded**: Slayte runs within Impexium as a true application, indivisible to your staff from any other Impexium application


**Capture Impexium Profile Fields in Slayte**: Slayte utilizes the Impexium API to its fullest and is able to capture a variety of fields from an individual's profile as part of their form submission.


**User Search**: Enable your users to find their co-authors from Impexium, or find nominations from your actual database, avoiding data duplication and siloing.


**Query Slayte from Impexium:** Slayte submissions and reviews are searchable through Impexium queries, and can be incorporated into Dashboards in Impexium via Custom Data Tables.


**Slayte - Impexium Write-Back**: Sessions created in Slayte can be exported as Sessions in an Impexium event, providing a full bi-directional integration.


## User Restrictions




**User must have purchased**: If you would like to restrict access to users who have previously purchased an item in Impexium, select this option.


**Member Category**: This allows members of a certain category to submit submissions using a specific Category Code.


**Member Status**: This decides whether you would like to restrict submission access to members or not. Select "Yes" to only allow members. Select "No" to enable non-members.


**Committee Position**: This allows users in at least one committee to have a certain position (position code).


**Region**: If you would only like to allow users from a certain geographic area, select this option.


**Relationship Type**: This option only allows users that have a specific relationship type to use the relationship type name.


**Event Registration**: This option only allows registered users for a particular event. A User will need to use an Event Code to be able to send in a submission.
